# Gorn Implementation Plan

This plan is intentionally small. The priority is to make one-file `.gorn` scripts run reliably before adding fragments, import inference, interactive shell behavior, or richer tooling.

---

## Initial repo layout

```text
gorn/
  go.mod
  cmd/
    gorn/
      main.go
  internal/
    directive/
    gornfile/
    generate/
    cache/
    runner/
  sh/
    result.go
    cmd.go
    args.go
    glob.go
    fs.go
    file.go
    path.go
  testdata/
    scripts/
      hello.gorn
      deps.gorn
      todo.gorn
    golden/
      hello.main.go
      hello.go.mod
```

Initial module path:

```text
module gorn.dev/gorn
```

Public package:

```go
import "gorn.dev/gorn/sh"
```

Generated `.gorn` code should import the same package.

## Phase 1: `sh` package v0

Build `sh` first because it is independently testable and gives Gorn scripts their value.

### Deliverables

```go
// Process.
func Cmd(name string, args ...any) Result
func Run(name string, args ...any) Result
func Spread(values ...string) ArgList

// Result.
type Result struct { /* ... */ }
func (r Result) OK() bool
func (r Result) Error() error
func (r Result) OrExit() Result
func (r Result) OrExitf(format string, args ...any) Result

// Failure helpers.
func OrExit(err error)
func OrExitf(err error, format string, args ...any)
func Fatal(args ...any)
func Fatalf(format string, args ...any)

// Glob/path.
func Glob(pattern string, opts ...GlobOption) []string
func MustGlob(pattern string, opts ...GlobOption) []string
func Home() string
func Expand(path string) string
func Abs(path string) string
func Join(parts ...string) string

// Files.
func Exists(path string) bool
func IsFile(path string) bool
func IsDir(path string) bool
func Mkdir(path string, opts ...MkdirOption) Result
func Remove(path string, opts ...RemoveOption) Result
func Copy(src, dst string, opts ...CopyOption) Result
func Move(src, dst string, opts ...MoveOption) Result

// Text files.
func Read(path string) (string, error)
func MustRead(path string) string
func ReadLines(path string) ([]string, error)
func MustReadLines(path string) []string
func Write(path, text string) Result
func Append(path, text string) Result
```

### Initial typed options

```go
// Remove.
RemoveForce
RemoveRecursive

// Mkdir.
MkdirParents

// Copy.
CopyOverwrite
CopyRecursive

// Move.
MoveOverwrite

// Glob.
GlobDotfiles
GlobFollowSymlinks
GlobUnsorted
GlobFailOnNoMatch
```

### Tests

- `Cmd` captures stdout/stderr.
- `Run` streams and returns status.
- `Spread` expands explicitly.
- Direct `[]string` in command args is rejected or panics with a clear message.
- `Result.OrExitf` formats command and native-helper failures.
- `Glob("**/*.go")` recurses and sorts.
- `GlobDotfiles` changes dotfile behavior.
- `Read`, `Write`, `Append` round-trip.
- `Copy`, `Move`, `Remove`, `Mkdir` work on Linux/macOS/Windows.

## Phase 2: `.gorn` parser

### v0 file shape

```go
#!/usr/bin/env gorn
//gorn:go 1.26
//gorn:require github.com/charmbracelet/lipgloss v1.1.0

import (
	"fmt"
	"strings"
)

func hasTodo(path string) bool {
	return strings.Contains(sh.MustRead(path), "TODO")
}

//gorn:main

for _, file := range sh.Glob("**/*.go") {
	if hasTodo(file) {
		fmt.Println(file)
	}
}
```

### Parse result

```go
type Script struct {
	Path           string
	Source         []byte
	GoVersion      string
	Module         string
	Requires       []Require
	PackageSection string
	MainSection    string
}

type Require struct {
	Path    string
	Version string
}
```

### Rules

- Optional shebang only on line 1.
- `//gorn:` directives are parsed before `//gorn:main`.
- Exactly one `//gorn:main` is required.
- Everything before `//gorn:main` is package section.
- Everything after `//gorn:main` is main section.
- Directives after `//gorn:main` are errors.
- `//gorn:require` must include an explicit version.

### v0 directives

```go
//gorn:go <version>
//gorn:module <module-path>
//gorn:require <module-path> <version>
//gorn:main
```

### Parser tests

- Parses shebang.
- Parses `go`, `module`, and `require` directives.
- Splits at `//gorn:main`.
- Errors on missing `//gorn:main`.
- Errors on multiple `//gorn:main` directives.
- Errors on invalid `//gorn:require`.
- Errors on directive after `//gorn:main`.

## Phase 3: code generation

Use `text/template` for generated files, then run `go/format`.

### Generated `go.mod`

Template:

```gotemplate
module {{ .Module }}

go {{ .GoVersion }}

{{- if .Requires }}
require (
{{- range .Requires }}
	{{ .Path }} {{ .Version }}
{{- end }}
)
{{- end }}
```

Default module if omitted:

```text
gorn.local/app/<short-source-hash>
```

### Generated Go

Template:

```gotemplate
// Code generated by gorn from {{ .SourcePath }}; DO NOT EDIT.

package main

import "gorn.dev/gorn/sh"

{{ .PackageSection }}

func main() {
{{ indent "\t" .MainSection }}
}
```

Then:

```go
formatted, err := format.Source(generated)
```

Multiple import declarations are valid. Do not attempt import hoisting or alias rewriting in v0.

### Validation

After generation and formatting, parse the generated source:

```go
_, err := parser.ParseFile(fset, generatedPath, formatted, parser.ParseComments)
```

Return generated-source diagnostics at first. Source mapping can improve later.

### Optional source mapping

Later, add `//line` directives around package and main sections so compiler errors point back to the `.gorn` file.

## Phase 4: uncached runner

Before building the cache, make this work with a temporary directory:

```text
gorn run hello.gorn
```

Flow:

1. Read `.gorn` source.
2. Parse directives and split sections.
3. Generate temp module directory.
4. Write `go.mod`.
5. Write `main.gorn.go`.
6. Run `go mod tidy` if needed.
7. Run `go build -o <temp-bin>`.
8. Execute the binary with forwarded args.
9. Remove temp directory unless `--keep` is specified.

Useful flags:

```text
--keep
--print-gen
--print-mod
```

This phase proves the source format and generator before cache complexity is added.

## Phase 5: cache

Gorn should use the Go build and module caches rather than replacing them. Gorn's cache stores the generated app project and final binary.

### Location

Use `os.UserCacheDir()`.

Typical paths:

```text
Linux:   ~/.cache/gorn/
macOS:   ~/Library/Caches/gorn/
Windows: %LocalAppData%\gorn\Cache\
```

Layout:

```text
<cache>/gorn/
  apps/
    <cache-key>/
      manifest.json
      go.mod
      go.sum
      main.gorn.go
      bin/
        app-or-app.exe
  tmp/
  locks/
```

### Cache key inputs

Include:

- Gorn version.
- Generator/cache schema version.
- Absolute source path.
- Source bytes.
- Parsed directives.
- Generated `go.mod` bytes.
- Go toolchain version.
- `GOOS`.
- `GOARCH`.
- Build tags, if supported.

Do not include:

- Runtime args.
- Current working directory.
- Runtime environment.
- Files read by the script at runtime.
- External commands called by the script.

### Build flow

```text
1. Compute cache key.
2. If cache entry has valid binary + manifest, run binary.
3. Otherwise acquire per-key lock.
4. Re-check cache after lock.
5. Generate/build in temporary directory.
6. Atomically move completed output into cache.
7. Write manifest.
8. Run binary.
```

Never run a partially built cache entry.

### Manifest

```json
{
  "schema": 1,
  "gorn_version": "0.0.1",
  "go_version": "go1.26.0",
  "goos": "linux",
  "goarch": "amd64",
  "source": "/home/user/scripts/todo.gorn",
  "source_hash": "abc123...",
  "module": "gorn.local/app/abc123",
  "requires": [
    {
      "path": "github.com/charmbracelet/lipgloss",
      "version": "v1.1.0"
    }
  ],
  "built_at": "2026-06-30T21:42:00Z"
}
```

### Cache commands

```text
gorn cache path
gorn cache list
gorn cache clean
gorn cache prune
```

Optional later:

```text
gorn cache inspect <script.gorn>
```

## Phase 6: CLI

Use ordinary Go flag parsing first unless a real need for a CLI framework appears.

Initial commands:

```text
gorn run <script.gorn> [-- args...]
gorn build <script.gorn> -o <path>
gorn cache path
gorn cache clean
gorn version
```

Build flags:

```text
--rebuild       ignore Gorn binary cache
--no-cache      build in temp dir and discard output
--keep          keep generated project and print path
--print-gen     print generated Go
--print-mod     print generated go.mod
```

Runtime args:

```text
gorn run script.gorn -- arg1 arg2
```

Also allow, if not ambiguous:

```text
gorn run script.gorn arg1 arg2
```

## Phase 7: dependency support

Support explicit requirements:

```go
//gorn:require github.com/charmbracelet/lipgloss v1.1.0
```

Do not support `latest` in source.

Later helper:

```text
gorn add github.com/charmbracelet/lipgloss@latest script.gorn
```

This command can resolve latest once and write a concrete version into the file.

Defer `//gorn:replace` until needed.

---

## Phase 8: diagnostics and source maps

Initial diagnostics can show generated Go errors. This is acceptable for early development but should be improved before broader release.

Future improvement:

- Insert `//line` directives so compiler errors map to `.gorn` lines.
- On generation failure, keep generated source with `--keep`.
- Provide `--print-gen` for debugging.
- Add friendly errors for common Gorn mistakes:
  - missing `//gorn:main`
  - multiple `//gorn:main`
  - invalid directive syntax
  - dependency missing version
  - directive after `//gorn:main`

`sh` package:

- process execution
- result formatting
- explicit spread behavior
- glob behavior

- text file helpers

`internal/gornfile`:

- directive parsing
- section splitting

- `go.mod` generation

- formatting

`internal/cache`:

- cache key stability
- cache key invalidation
- manifest read/write

### Integration tests

Run real scripts:



## Milestone sequence

### Milestone 1: `sh` works in ordinary Go

No `.gorn` runner yet. Prove the helper API.

### Milestone 2: generate and print

```text
gorn run --print-gen hello.gorn
```

No build yet.

### Milestone 3: temp build/run

```text
gorn run hello.gorn
```

Generate temp module, build, run.

### Milestone 4: cache

Add content-addressed cache and fast reruns.

### Milestone 5: dependencies

Support `//gorn:require` and generated `go.mod`.

### Milestone 6: `build`

```text
gorn build script.gorn -o ./script
```

### Milestone 7: diagnostics

Improve generated/source error mapping and friendly Gorn errors.

---

## Deferred features

Do not build these in v0:

- `.gfrag` fragments.
- `//gorn:include` or `//gorn:fragment`.
- Automatic import insertion.
- Import hoisting or alias rewriting.
- Interactive shell mode.
- Native `grep`, `find`, `sed`, `awk`, `tar`, etc.
- Pipeline DSL.
- Argument parser in `sh`.
- Remote scripts.
- `latest` dependency directives.
- Watch mode.
- LSP.
- Formatter for `.gorn` source.
- `gorn eject`.

These are allowed future directions, but they should not block the first useful version.

```go
#!/usr/bin/env gorn

import (
	"fmt"
	"strings"
)

//gorn:main

for _, file := range sh.Glob("**/*.go") {
	if strings.Contains(sh.MustRead(file), "TODO") {
		fmt.Println(file)
	}
}
```

Run:

```sh
gorn run todo.gorn
```

This proves:

- `.gorn` parsing.
- Explicit `//gorn:main` split.
- Generated Go module.
- Automatic `sh` import.
- `sh.Glob`.
- `sh.MustRead`.
- Go stdlib use.
- Build/run path.
- Cache reuse.
